# Contoso Market Research Secure Agent

Secure hosted-agent variant for the Contoso market research scenario.

This app keeps the controlled PostgreSQL data-access flow, but uses Foundry's integrated Code Interpreter for charts instead of a custom code-execution tool inside the hosted agent container.

## Architecture

![Architecture diagram](./safe_architecture.png)

| Resource | Purpose |
| --- | --- |
| AI Services + Project | Foundry project for the hosted market research agent |
| ACR | Hosts the secure agent container image |
| PostgreSQL Flexible Server | Contoso database with fake customers, sales, and employee compensation data |
| Foundry Code Interpreter | Managed sandbox for chart generation and analysis from sanitized data |
| Storage Account | Private short-lived chart artifact delivery for hosted-agent UI compatibility |

## Security model

The hosted container receives `DATABASE_URL` so the custom `get_market_data` tool can query PostgreSQL through sanitized SQL views. Generated Python does **not** run in that container. Chart creation happens in Foundry's integrated Code Interpreter, which should only receive sanitized data returned by `get_market_data`.

Visual outputs are generated by Code Interpreter, downloaded by the trusted hosted adapter with managed identity, uploaded to a private blob container, and returned as short-lived read-only SAS URLs. This storage step is artifact delivery only; it does not reintroduce custom code execution.

## Setup

```bash
azd up
```

`azd up` will:

1. Provision Azure resources: PostgreSQL, AI Services, ACR, and private chart storage.
2. Run post-provision setup:
   - Open a PostgreSQL firewall rule for your current public IP.
   - Seed PostgreSQL with fake Contoso data.
   - Build the secure hosted agent image in ACR.
   - Deploy and start the `contoso-market-research-secure` hosted agent.
   - Generate `src/agent/.env` for local development.

If `azd up` halts during postprovision, re-run:

```bash
azd hooks run postprovision
```

## The agent

The **Contoso Market Research Secure Agent** is a single hosted ChatAgent. Its normal flow is:

1. Call `get_market_data` to retrieve sanitized data from one of the allowed datasets.
2. Use Foundry Code Interpreter to create a chart from that sanitized data.
3. Return a concise summary and a `Chart URL:` pointing to the generated visual.

## Local agent development

```bash
cd src/agent
# .env is generated automatically by setup.py during azd up
python main.py
# Agent server runs on http://localhost:8088
```
